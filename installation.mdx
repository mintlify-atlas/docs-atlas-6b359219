## System Requirements

Before building Jdeflate, ensure you have the following installed:

### Required

- **C99-compliant compiler** (GCC, Clang, MSVC, or compatible)
- **Python 3.7 or newer**
- **Meson build system** (0.55.0 or newer)
- **Ninja build system**

### Optional

- **NASM** (Netwide Assembler) - For optimized x86-64 assembly implementations of CRC32 and Adler32 checksums

<Note>
If NASM is not available, Jdeflate will automatically fall back to C implementations. The library will work correctly either way, but assembly versions offer better performance.
</Note>

## Building from Source

<Steps>

<Step title="Clone the repository">

First, get the Jdeflate source code:

```bash
git clone https://github.com/Jpn666/jdeflate.git
cd jdeflate
```

</Step>

<Step title="Configure the build">

Create a build directory and configure the project:

```bash
meson setup builddir
```

This creates a `builddir` directory with the build configuration. Meson will automatically detect your compiler and configure optimal settings.

<Note>
The default build type is `debug`. For production use, you'll want to configure a release build (see Build Options below).
</Note>

</Step>

<Step title="Compile the library">

Build the project using Ninja:

```bash
meson compile -C builddir
```

This compiles the library and places the output in `builddir/`.

</Step>

<Step title="Install (optional)">

To install the library system-wide:

```bash
sudo meson install -C builddir
```

This installs:
- Library files to the appropriate lib directory
- Header files to the include directory
- pkg-config files for easy integration

<Warning>
On most systems, this requires root/administrator privileges.
</Warning>

</Step>

</Steps>

## Build Options

You can customize the build using Meson's configuration options:

### Library Type

Choose between static or shared library:

<CodeGroup>

```bash Static Library
meson setup builddir --default-library=static
```

```bash Shared Library
meson setup builddir --default-library=shared
```

```bash Both
meson setup builddir --default-library=both
```

</CodeGroup>

<Note>
The default is `shared`. When building as a shared library on Windows (MSVC), consumers must define `JDEFLATE_DLL` when including Jdeflate headers.
</Note>

### Build Type

Control optimization and debug symbols:

<CodeGroup>

```bash Release
# Optimized for production use
meson setup builddir --buildtype=release
```

```bash Debug
# No optimization, full debug symbols
meson setup builddir --buildtype=debug
```

```bash Debug Optimized
# Balanced: optimizations + debug symbols
meson setup builddir --buildtype=debugoptimized
```

```bash Plain
# No special flags
meson setup builddir --buildtype=plain
```

</CodeGroup>

### Combining Options

You can combine multiple options:

```bash
meson setup builddir \
  --default-library=static \
  --buildtype=release \
  --prefix=/usr/local
```

### Reconfiguring an Existing Build

To change options on an existing build directory:

```bash
meson configure builddir --default-library=static --buildtype=release
meson compile -C builddir
```

## Using as a Meson Subproject

Jdeflate can be integrated into your Meson project as a subproject:

<Steps>

<Step title="Create subprojects directory">

```bash
mkdir -p subprojects
```

</Step>

<Step title="Add Jdeflate as a subproject">

Create `subprojects/jdeflate.wrap`:

```ini
[wrap-git]
url = https://github.com/Jpn666/jdeflate.git
revision = head
depth = 1
```

</Step>

<Step title="Use in your meson.build">

```python
project('myproject', 'c')

# Get Jdeflate dependency
jdeflate_dep = dependency('jdeflate', 
  fallback: ['jdeflate', 'jdeflate_dep']
)

# Use in your executable or library
executable('myapp',
  'src/main.c',
  dependencies: jdeflate_dep
)
```

</Step>

</Steps>

Meson will automatically download and build Jdeflate as part of your project's build process.

## Using with pkg-config

After installing Jdeflate, you can use pkg-config to get compiler and linker flags:

```bash
# Get compiler flags
pkg-config --cflags jdeflate

# Get linker flags
pkg-config --libs jdeflate

# Check installed version
pkg-config --modversion jdeflate
```

### Example: Manual Compilation

```bash
# Compile and link manually
gcc myapp.c $(pkg-config --cflags --libs jdeflate) -o myapp
```

### Example: In a Makefile

```makefile
CFLAGS = $(shell pkg-config --cflags jdeflate)
LDFLAGS = $(shell pkg-config --libs jdeflate)

myapp: main.o
	$(CC) -o $@ $^ $(LDFLAGS)

main.o: main.c
	$(CC) -c $(CFLAGS) $< -o $@
```

## Platform-Specific Notes

### Windows (MSVC)

When using Jdeflate as a shared library (DLL) on Windows with MSVC, you must define the `JDEFLATE_DLL` macro:

```c
#define JDEFLATE_DLL
#include <jdeflate/deflator.h>
#include <jdeflate/inflator.h>
```

Or pass it as a compiler flag:

```bash
cl /DJDEFLATE_DLL myapp.c jdeflate.lib
```

<Warning>
Failure to define `JDEFLATE_DLL` when using the shared library will result in linker errors.
</Warning>

### x86-64 Assembly Optimizations

On x86-64 systems, Jdeflate can use optimized assembly implementations for CRC32 and Adler32 checksums:

- **Supported on**: Windows, Linux, macOS, BSD variants
- **Requires**: NASM assembler
- **Automatic**: If NASM is detected during build, assembly versions are used automatically
- **Fallback**: If NASM is not available, C implementations are used instead

You can verify if assembly optimizations are enabled by checking the build output:

```bash
meson setup builddir
# Look for: "Program nasm found: YES"
```

## Verifying Installation

After installation, verify that Jdeflate is correctly installed:

```bash
# Check if headers are accessible
ls /usr/local/include/jdeflate/

# Verify pkg-config can find it
pkg-config --exists jdeflate && echo "Jdeflate found"

# Display version
pkg-config --modversion jdeflate
```
